定时脚本，顾名思义，是指在特定时间或时间间隔自动执行的脚本。在 Quantumult X 中，用户可以通过配置定时任务，让脚本在后台按计划运行。这种自动化能力为用户打开了无限可能：

  3. 监控与告警：通过定时检查网络连接或服务状态，及时发现问题并通知用户。

这些应用场景不仅节省了用户的时间，还减少了因人为疏忽导致的错误。更重要的是，定时脚本允许用户根据自身需求定制功能，真正实现个性化网络管理。

要使用 Quantumult X 的定时脚本功能，用户需要具备基本的 JavaScript 知识。以下是详细的步骤指南：

FAQ

VLESS + XTLS 在移动网络的优势是什么？
XTLS 模式减少握手次数和加密开销，适合高延迟或不稳定移动网络。结合策略组和节点切换，可提升网页浏览、视频播放和游戏连接的稳定性和流畅度。
如何配置 Sing-box 使用 DoH 加密 DNS？
在 dns 模块中添加类型为 doh 的上游，如 `https://1.1.1.1/dns-query`，并指定 bootstrap 地址。启用加密 DNS 可防止 DNS 泄露并加快国际解析速度，提高整体隐私保护。
Clash 的 Proxy Group 能实现哪些策略？
Proxy Group 可以定义多种策略，如按延迟选择、随机、轮询或手动选择节点。结合分流规则，可实现针对不同网站或应用的智能代理，提高网络体验。
VLESS + XTLS 模式能减少握手延迟吗？
XTLS 通过减少握手次数和加密开销，降低延迟，特别适合短连接或高并发场景。结合策略组，可保证连接稳定性和访问速度，适用于移动网络和游戏环境。
Clash 的延迟检测如何帮助节点选择？
Clash 会定期测速节点延迟，并根据策略组自动选择延迟最低节点处理流量。可避免访问高延迟或失效节点，提高网页、视频和游戏访问速度及稳定性。
PassWall2 的延迟测速功能如何提高网络体验？
PassWall2 可以定期测速节点延迟，并结合策略组自动选择最优节点。这样在访问网站或视频时，可以始终使用延迟最低的线路，提高网络稳定性和速度。
为什么使用 CDN 后仍然能被封？如何规避？
若 CDN 回源 IP 或配置不当可能泄露真实服务器信息，或者 CDN 本身被目标防火墙识别。应启用完整的反向代理配置、隐藏源站 IP 与禁用不必要的服务，必要时使用多级中转或更换 CDN 供应商。
SSR 协议和混淆组合如何优化稳定性？
根据网络封锁情况选择合适协议和混淆组合，如 tls1.2_ticket_auth 模拟 HTTPS 流量。合理组合可提高抗封锁能力和稳定性，保证节点长期可用并优化网络连接。
Nekoray 的内置延迟测试为什么与系统 ping 不同？
客户端的延迟测试通常基于 TCP 或特定探测包，而系统 ping 使用 ICMP，两者网络路径与优先级可能不同。此外，防火墙或服务器对 ICMP 的响应可能被限制，导致测试结果存在差异。
Quantumult X 可以用脚本自动切换节点吗？
可以。Quantumult X 支持通过脚本逻辑判断网络延迟或连接失败情况，并自动切换节点。用户可以编写自定义 JS 规则，从而实现智能代理与按需分流，适合高级用户使用。
