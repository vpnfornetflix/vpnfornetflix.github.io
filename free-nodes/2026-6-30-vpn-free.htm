二、Quantumult核心功能全面解读

1. 代理节点管理

Quantumult最基础也是最重要的功能便是代理连接。它支持导入来自机场的订阅链接，也可手动配置代理节点信息。

  • 支持协议：VMess、SS、SSR、Trojan、HTTP(S)、Socks5

  • 节点分组管理：支持将节点分组，根据需求灵活切换

  • 节点测速功能：可检测节点响应速度，优化使用体验

2. 策略组与规则系统

Quantumult最强大的模块之一就是其策略组与规则引擎。

  • 策略组：你可以将多个节点打包成一个“策略组”，如“国内”、“国际”、“流媒体专用”等，根据不同规则自动选择路径。

  • 规则匹配：可使用默认规则集（如广告过滤、地区分流），也可以导入第三方规则库，支持GeoIP、Domain、IP-CIDR等方式。

  • 自动分流：例如，将Netflix、Disney+流量分流至支持解锁的节点，其余走直连或普通代理。

3. 重写功能（Rewrite）

Quantumult的Rewrite功能支持通过修改请求头、响应体等方式，实现广告过滤、页面精简、请求重定向等高级操作。

  • 常见用途：微博去广告、知乎去推荐、B站防启动广告

  • 支持脚本：兼容Quantumult X格式，可导入JavaScript脚本实现更多自定义逻辑

4. 流量统计与日志分析

量化网络体验是优化配置的第一步。

  • 实时流量监控：查看当前代理流量和直连流量使用情况

  • 历史统计：按天/周/月记录使用数据，清晰掌握带宽消耗

  • 详细日志输出：便于调试规则或排查连接问题

5. 系统集成与快捷功能

FAQ

PassWall2 的延迟测试如何影响策略组选择？
PassWall2 会根据节点延迟结果更新策略组的优先节点，自动选择延迟最低的节点处理流量。这样可以优化访问速度，提高网页浏览、视频和游戏体验的稳定性。
Mellow 的 TUN 模式适合哪些应用？
TUN 模式拦截系统 IP 层流量，可实现全局代理。适合游戏、视频、浏览器和其他全局应用，实现低延迟和稳定连接，保证所有流量通过代理节点，提高访问速度和稳定性。
Sing-box 如何按 IP 段实现智能分流？
Sing-box 支持根据 IP 段匹配流量，将国内流量直连、国外流量走代理。结合策略组和规则文件，实现精细化流量管理，保证访问速度和稳定性，同时提高节点使用效率。
SSR 还能用吗？
虽然 SSR 项目早已停止维护，但部分服务器和机场仍提供 SSR 节点。SSR 使用的加密与混淆方式能应对部分防火墙检测。不过，建议用户逐步迁移到 V2Ray 或 Trojan 等更现代的协议以获得更好稳定性。
Matsuri 可以用于 Android 吗？
Matsuri 主要针对桌面平台开发，目前并未推出 Android 版本。但可使用其配置文件在兼容的客户端（如 Nekobox 或 Clash Android）中加载以获得类似体验。
PassWall2 如何结合策略组实现自动节点切换？
PassWall2 会定期测试节点延迟和丢包率，结合策略组自动切换到最优节点。无需手动干预即可保证网页、视频和游戏流量高速稳定访问，优化整体网络体验。
WinXray 是否支持 VMess + TLS + WS 模式？
是的，WinXray 支持该组合。用户需填写正确的 host、path、端口及证书验证方式。若节点来自订阅源，将自动识别相应参数；否则需手动输入与服务端一致的配置项。
Trojan TLS 如何避免被 DPI 检测？
通过选择真实可访问域名并使用有效证书，Trojan 流量可完全模拟 HTTPS 请求。隐藏真实流量特征，降低被封锁和 DPI 检测风险，保证节点长期可用。
Mellow 是否支持自定义 DNS？
支持。Mellow 的配置文件中可设置 DNS 服务器地址及模式（如 DoH、DoT）。用户可使用安全 DNS，如 Cloudflare 1.1.1.1 或 Google DNS，以减少污染风险。
如何用 iptables 限制 V2Ray 服务的访问来源？
使用 iptables 规则仅允许可信 IP 或 CDN 的中转 IP 访问服务端口，例如 `iptables -A INPUT -p tcp -s --dport 443 -j ACCEPT`，并在默认策略下拒绝其他来源，从而降低被扫描与滥用风险。
